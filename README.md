# Grocery Store Management System (SQL Project)

## Project Overview
This project focuses on designing and managing a Grocery Store Management System using SQL. The database stores and organizes information related to customers, products, suppliers, employees, and orders. SQL queries are used to analyze sales trends, customer behavior, product performance, and employee activity.

## Database Tables
The database contains the following tables:

- Supplier
- Categories
- Employees
- Customers
- Products
- Orders
- Order_Details

These tables are connected using primary keys and foreign keys to maintain data integrity.

## Analysis Performed

### Customer Insights
- Unique customers placing orders
- Customers with the highest number of orders
- Total and average purchase value per customer
- Top customers by total purchase value

### Product Performance
- Number of products in each category
- Average price of products by category
- Products with the highest sales quantity
- Total revenue generated by each product

### Sales and Order Trends
- Total number of orders
- Average order value
- Orders placed by date
- Monthly sales trends

### Supplier Contribution
- Number of suppliers in the database
- Suppliers providing the most products
- Supplier revenue contribution

### Employee Performance
- Employees handling the most orders
- Total sales processed by each employee
- Average order value handled per employee
